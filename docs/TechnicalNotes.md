# TechnicalNotes.md

## Development Approach

Use the current Getnetic website/codebase as a starting point where possible.

Do not rebuild everything from scratch unless necessary.

Prefer adapting existing layout, animation, components, form logic, privacy policy, and cookie consent.

## MVP Scope

The MVP should be lightweight.

Required:
- One main landing page
- Contact section
- Privacy policy
- Cookie banner if already present
- Responsive layout
- Basic SEO metadata

Not required for MVP:
- Blog
- CMS
- Case studies
- Full service pages
- Multi-language setup
- Advanced animations
- Complex backend

## Contact Form

Reuse existing form logic if possible.

Check:
- Recipient email
- Spam protection
- Validation
- Success message
- Error message

## Privacy / Cookies

Reuse existing:
- Privacy Policy
- Cookie banner
- Cookie consent behavior

Do not remove legal/compliance-related pages unless explicitly approved.

## Media

Hero video should be optimized.

Use:
- compressed mp4/webm
- fallback image
- lazy loading where possible
- mobile-friendly media behavior

## Performance

Important:
- Compress images
- Avoid huge background videos
- Minimize unused scripts
- Keep animations smooth
- Test mobile performance

## Accessibility

Minimum:
- Keyboard navigation
- Visible focus states
- Proper contrast
- Alt text for images
- Semantic headings
- Form labels

## SEO Basics

MVP title idea:
Getnetic — Field Engineering for Hard-to-Access Infrastructure

Meta description:
Industrial diagnostics, inspections, and field engineering support for critical environments and hard-to-access infrastructure.
