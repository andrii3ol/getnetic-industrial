## General Rules

Read ProjectContext.md first.

Then read:
- Content.md
- References.md
- InformationArchitecture.md
- VisualDirection.md
- TechnicalNotes.md
- Tasks.md

Do not start coding before understanding the project direction.

## Project Direction

This is not a generic engineering company website.

This is a modern industrial-tech landing page for field engineering and hard-to-access infrastructure.

## Main Headline

Always preserve the main headline unless explicitly asked to change it:

Field Engineering for Hard-to-Access Infrastructure

## Tone of Voice

Keep copy short, strong, and confident.

Preferred phrases:
- Built for critical environments
- Access. Inspect. Diagnose.
- Field-proven diagnostics
- Real-world engineering operations
- Engineered for hazardous environments
- Rapid industrial response

Avoid:
- Innovative solutions
- Cutting-edge technology
- We are passionate
- We empower businesses
- Generic SaaS/startup language
- Long corporate paragraphs

## Design Rules

Keep the design:
- dark
- industrial
- premium
- minimal
- cinematic
- technical

Avoid:
- corporate blue gradients
- generic stock office photos
- startup-style AI visuals
- overcomplicated sections
- too much text

## Code Rules

Prefer editing existing code over rebuilding from scratch.

Preserve:
- existing good animations
- responsive behavior
- contact form logic
- privacy policy
- cookie banner
- analytics setup if present

Do not:
- remove existing functionality without confirmation
- overwrite production website
- change live getnetic.com
- invent fake certifications
- invent fake case studies
- invent fake clients

## Design Ambition

Do not simply modernize the old Getnetic website.

Treat this as a next-generation 2026 industrial-tech product.

Target impression:

Premium.
Technical.
Cinematic.
Minimal.

Think:

Industrial technology platform.

Not:

Engineering services brochure.

The result should feel more advanced than the current Getnetic website.

Reuse existing code where practical, but do not reuse outdated design patterns simply because they already exist.

## MVP Rules

This is a concept validation MVP.

The goal is to impress stakeholders and validate direction.

Do not overbuild.

Focus on:
- hero impact
- clear services
- strong visual direction
- contact conversion
- clean mobile layout
