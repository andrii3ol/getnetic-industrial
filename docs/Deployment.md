# Deployment.md

## Current Situation

The existing getnetic.com website is live and should not be replaced during MVP development.

The new version should be developed separately and only replace the current website after stakeholder approval.

## Preferred Staging Options

Option 1:
new.getnetic.com

Option 2:
getnetic.com/new

Preferred:
new.getnetic.com

Reason:
- Cleaner staging environment
- Easier to present
- Easier to keep separate from current live site
- Safer for testing

## Deployment Rules

- Do not overwrite live getnetic.com
- Do not remove old project files
- Do not change DNS without confirmation
- Do not replace production until approved by Yegor Kozlov
- Keep MVP clearly separated from the current website
